Testing the sign-up functionality. Sign-up test scenario with valid and invalid inputs.

Pre Condition
- Use a new private browsing window for each test case. Depending on the Browser used, may be named as Private, Incognito or InPrivate.

- Only one GitLab test user account is required by further tests. Delete any other extra GitLab user accounts created during this test.

#******************** Test Cases *************************

*** Test Cases ***

# Positive Tests

Sign Up With All Valid Variables
- Navigate to SignUpUrl.
- Input first and last name.
- Input username A. The page should display "Username is available" in green text.
- Input email A.
- Input valid password. The page should display the three conditions for a valid password in green text.
- Click the Continue button on the page.
- Input the verification code that was sent to the valid email address.
- Click the verify email address button.
- The Welcome page asks for further input, but those are not relevant to this test case.

# Negative Tests

Sign Up With Invalid Username
- Navigate to SignUpUrl.
- Input first and last name.
- Input username A. The page should display "Username is already taken" in red text.
- Input email B.
- Input valid password. The page should display the three conditions for a valid password in green text.
- Click the Continue button on the page.
- Verify that the page displays following message at the top: 
1 error prohibited this user from being saved:
Username has already been taken

Sign Up With Invalid Email
- Navigate to SignUpUrl.
- Input first and last name.
- Input username B. The page should display "Username is available" in green text.
- Input email A.
- Input valid password. The page should display the three conditions for a valid password in green text.
- Click the Continue button on the page.
- Verify that the page displays following message at the top: 
1 error prohibited this user from being saved:
Email has already been taken

Sign Up With Invalid Password
- Navigate to SignUpUrl.
- Input first and last name.
- Input username B. The page should display "Username is available" in green text.
- Input email B.
- Input invalid password. The page should display two of the three conditions for a valid password in green text.
- Click the Continue button on the page.
- Verify that the page moves cursor to the password field, highlights the field and displays one of the three conditions for a valid password as red.
